# 第5课设计文档：AI 的调色板——生成式AI的秘密

**日期**：2026-04-20  
**目标学员**：安安（9岁，有Scratch基础）  
**输出文件**：`ai-for-anan-5.html`（单文件，与现有课程同架构）

---

## 一、教学目标

1. **理解前沿概念**：安安能用自己的话说清楚"生成式AI是怎么工作的"
2. **激发编程兴趣**：让她看到 Scratch 里已经在用的逻辑（随机数、规则、循环）和 AI 创作是同一件事的不同规模
3. **核心认知翻转**：创造力不是魔法，是数学——而数学是程序员的语言

---

## 二、课程结构（6张卡片 + 测验 + 总结）

### Card 1 · 开场谜题「你能看出来吗？」
- **内容**：展示4张图（AI生成 vs 人类创作混排），让安安猜哪张是AI画的
- **互动**：点击图片猜测，点击后翻转揭晓答案
- **目的**：制造认知冲击，激发好奇心，引出"AI是怎么做到的？"

### Card 2 · AI没有眼睛，但它有"感觉"
- **内容**：AI不存图片，存的是"模式"——线条、配色、形状组合代表的概念
- **类比**：你不用记住每首歌的每个音符，但你知道什么音乐"听起来悲伤"——AI也一样
- **互动**：词语联想图——展示一个2D概念地图，"猫"靠近"爪子""毛茸茸""喵"，拖拽词语看它落在哪里
- **编程钩子**：这叫"向量"——把意思变成数字，数字就能计算

### Card 3 · 从噪点到图像——wow时刻
- **内容**：扩散模型（Diffusion Model）核心思想——从随机噪点一步步"去噪"，浮现出图像
- **类比**：像在暗房里冲洗照片，图像从雾气里慢慢显现
- **互动**：
  - Canvas动画展示去噪过程（预生成约20帧静态图，用滑块/按钮控制进度）
  - "步数"滑块：拖到0是纯噪点，拖到100是清晰图像，中间可看半成品
  - 提供2-3个不同主题（小猫、宇宙飞船、彩虹蛋糕）
- **重点**：让她停在第50步看"幽灵图"，体验AI"脑中的模糊想象"

### Card 4 · 你来当提示工程师
- **内容**：Prompt的组成 = 变量的组合
- **互动**：卡片组合器
  - 三行选项卡：主题（猫/龙/宇航员）× 风格（水彩/像素/油画）× 心情（开心/神秘/壮观）
  - 组合后生成对应的"模拟Prompt"文本 + 对应描述性结果文字（不需要真实调用AI）
- **编程钩子**（明确点出）：
  > "你刚才做的事，程序员叫它「传参数」——就是Scratch里拖进积木的那些数字和颜色"
- **连接第3课**：这就是提示工程在生成式AI里的样子

### Card 5 · 从Scratch到AI：一步步
- **内容**：打通 Scratch → Python → AI 的概念进化链
- **视觉**：横向时间线/进化图，每个节点展开说明：
  - Scratch `随机数(1到10)` → Python `random.randint(1,10)` → AI的随机噪点
  - Scratch `如果…那么` → Python `if...else` → AI的决策模式
  - Scratch `重复10次` → Python `for i in range(10)` → AI的训练迭代
- **结语文案**（重要）：
  > "你在Scratch里做的事，和AI做的事是同一件事——只是规模不同。程序员用数字创造艺术。这就是你可以成为的人。"
- **互动**：点击每个Scratch积木，展开对应的Python代码和AI概念

### Card 6 · 生成式AI能创造什么？
- **内容**：6个生成式AI应用领域展示
  - 🖼️ 图像（Midjourney / DALL-E）
  - 🎵 音乐（Suno）
  - 📖 故事（Claude / GPT）
  - 🎬 视频（Sora）
  - 💻 代码（Claude Code / Copilot）
  - 🎮 游戏（AI生成关卡）
- **每个领域**：简短说明 + "你也能参与"的角度（比如：音乐→你可以用Suno给你的Scratch游戏配乐）
- **收尾**：呼应编程目标——"这些都是程序员在创造的东西"

## 三、测验（5题）

覆盖核心概念：
1. 扩散模型从什么开始生成图像？（随机噪点）
2. AI的"感觉"用什么表示？（数字/向量）
3. Prompt里的"主题+风格+心情"对应编程里的什么？（参数/变量）
4. Scratch的随机数积木和AI生成的关系？
5. 判断题：生成式AI是在"回忆"它见过的图片（错，它是在重新生成）
